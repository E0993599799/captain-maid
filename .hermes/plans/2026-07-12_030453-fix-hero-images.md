# Captain Maid Hero Image Fix Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Make the Captain Maid homepage hero carousel images render reliably on all breakpoints.

**Architecture:** Keep the current `CaptainMaidLandingPage` structure. The issue is not the assets themselves; it is the image wrapper sizing around `next/image` with `fill`. The fix should give each hero image container an explicit size/aspect ratio so the browser can reserve layout space and the images can paint. Do not rewrite the carousel or change the asset set.

**Tech Stack:** Next.js App Router, React, TypeScript, `next/image`, Tailwind CSS, Framer Motion.

---

## Current state

Confirmed from review:
- `app/[locale]/page.tsx` imports `CaptainMaidLandingPage` and is not the source of the render bug.
- `components/CaptainMaidLandingPage.tsx` renders the 5 hero slides with `Image fill` inside `figure` wrappers.
- The `figure` elements currently do not establish a fixed height or aspect ratio, so `fill` images can collapse to zero height.
- Static product images lower on the page already use explicit `width` / `height` and are not the main risk.

## Plan

### Task 1: Give each hero image wrapper a real box

**Objective:** Ensure every carousel slide image has a predictable rendered area.

**Files:**
- Modify: `components/CaptainMaidLandingPage.tsx`

**What to change:**
- Add an explicit aspect ratio or min-height to each `figure` that contains a `next/image` with `fill`.
- Keep the existing `relative overflow-hidden` pattern.
- Prefer one reusable class pattern so all 5 slide images behave consistently.

**Implementation notes:**
- Use Tailwind classes like `aspect-[4/3]`, `aspect-[16/10]`, or a responsive `min-h-[...]` depending on the existing composition.
- If the slide artwork is intended to be full-bleed, keep `fill` and control fit with `object-cover` / `object-contain`.
- Do not change the carousel timing, slide selection logic, or copy.

**Verification:**
- Check that the hero slide now occupies space before the image loads.
- Confirm no slide collapses to zero height at desktop, tablet, or mobile widths.
